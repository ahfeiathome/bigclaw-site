### Release Pipeline — Redesign

**Data source:** Change from parsing PRD_CHECKLIST.md markdown to reading `docs/product/verification-status.json` (generated by `scripts/generate-verification-json.py`). Fallback: parse RELEASE_VERIFICATION.md directly.

**Collapsed view — ONE LINE per release, three traffic lights:**

```
[color] vX.Y.Z "name" (N PRDs)   V-G [icon]  V-C [icon]  V-M [icon]   [Action Button]
```

Icons:
- ✅ = all testable items pass
- ⏳ = pending (not started)
- ❌ = at least one failure
- N/A = no items to test (Cat C excluded)

Color logic:
- 🟢 = V-G ✅ + V-C ✅ (ready for Michael, or fully shipped)
- 🟡 = V-G or V-C incomplete
- 🔴 = any FAIL
- ⬜ = blocked on external gate

Action button:
- "Review on Phone" → links to preview URL (when V-G ✅ + V-C ✅)
- "Waiting on CI" (when V-G incomplete)
- "Waiting on Review" (when V-C incomplete)
- "Blocked: [reason]" (when external gate)
- "✅ Live" (when deployed to production)
